◆ soTraversePath()

uint16_t sofs20::soTraversePath ( char *  path)

Get the inode associated to a given path.

The directory hierarchy of the file system is traversed to find an entry whose name is the rightmost component of the given path

See also
soGetDirentry
Parameters
[in]paththe path to be traversed
Precondition
path is an absolute path
path does not contain symbolic links
Exceptions
ENOTDIRif one of the path components, with the exception of the rightmost one, is not a directory
EACCESif the process that calls the operation does not have traverse (x) permission on all the components of the path, with exception of the rightmost one
Returns
the corresponding inode number
Note
When calling a function of any layer, the version with prefix so should be used, not the one with prefix grp or bin.

const char *  name 
)

Get the inode associated to a given name.

The directory contents, seen as an array of directory entries, is searched for an entry whose name is the given one.

Precondition
pih is a valid inode handler of a directory
name is a valid base name (it doesn't contains '/')
Parameters
[in]pihinode handler of the parent directory
[in]namethe name of the entry to be searched for
Returns
the corresponding inode number or InodeNullReference, if name doesn't exist
Note
When calling a function of any layer, the version with prefix so should be used, not the one with prefix grp or bin.

Add a new entry to the parent directory.

A direntry is added connecting the parent inode to the child inode.

Precondition
pih is a valid inode handler of a directory where the user has write access
cin is a valid inode number
name is valid and doesn't exist yet
Parameters
[in]pihinode handler of the parent inode
[in]namethe name of the entry to be created
[in]cininode number of the entry to be created
Note
The lnkcnt of the child inode is not to be changed by this function.
When calling a function of any layer, the version with prefix so should be used, not the one with prefix grp or bin.

◆ soDeleteDirentry()

uint16_t sofs20::soDeleteDirentry ( int  pih,
const char *  name 
)

Delete an entry from a parent directory.

The direntry associated from the given name is deleted.

  • If the entry is not the last one, the last one must be moved to the empty slot.
  • If the last data block becomes empty, it must be freed.
Precondition
pih is a valid inode handler of a directory where the user has write access
name is valid and exists
Parameters
[in]pihinode handler of the parent inode
[in]namename of the entry
Returns
the inode number of the deleted entry
Note
The lnkcnt of the child inode is not to be changed by this function.
When calling a function of any layer, the version with prefix so should be used, not the one with prefix grp or bin.
