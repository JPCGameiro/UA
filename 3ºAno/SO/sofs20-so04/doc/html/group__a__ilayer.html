uint16_t sofs20::soAllocInode ( uint32_t  mode)

Allocate a free inode.

An inode is retrieved from the list (bitmap) of free inodes and is properly initialized.

  • The ibitmap field of the superblock must be circularly searched and updated:
    • starting at the bit position following the one pointed to by the iidx superblock field,
    • looking for the first bit at zero.
    • putting it at one and returning its index, which corresponds to the inode number.
  • Fields mode, owner, and group of the allocated inode must be initialized (see unix system calls getuid and getgid).
  • The inode meta data fields in the superblock must be updated.
See also
https://www.gnu.org/software/libc/manual/html_node/Testing-File-Type.html
https://www.gnu.org/software/libc/manual/html_node/Permission-Bits.html
Precondition
type must represent either a file (S_IFREG), a directory (S_IFDIR), or a symbolic link (S_IFLNK);
permissions must represent a valid permission pattern (a octal value in the range 0000 to 0777);
Exceptions
ENOSPCif there are no free inodes
EINVALif type or permissions are invalid
Parameters
[in]modebitwise-or of type and permissions
Returns
the reference (number) of the inode allocated
Note
When calling a function of any layer, the version with prefix so should be used, not the one with prefix grp or bin.

void sofs20::soFreeInode ( uint16_t  in)

Free the referenced inode.

The inode is cleaned, marked as free, and inserted into the list of free inodes.

  • The corresponding bit of the ibitmap field is put at zero.
  • All inode fields are put at zero, except the references, that are put at BlockNullReference.
  • The inode meta data fields in the superblock must be updated.
Precondition
in must represent a valid inode number
Exceptions
EINVALif inode number is not valid.
Parameters
[in]innumber (reference) of the inode to be freed
Note
When calling a function of any layer, the version with prefix so should be used, not the one with prefix grp or bin.
