uint32_t sofs20::soGetFileBlock ( int  ih,
uint32_t  fbn 
)

Get the data block number corresponding to the given file block.

The file block number (fbn) is the number of a block from the file (inode) point of view, 0 representing the first block, 1 the second, and so forth.
The data block number (dbn) corresponding to fbn is given by d[fbn]. Note, however, that array d[.] is only partially stored in the inode.

Precondition
ih is a valid handler of an inode in-use
fbn is a valid file block number
Exceptions
EINVALif fbn is not valid
Parameters
[in]ihinode handler
[in]fbnfile block number
Returns
the number of the corresponding data block
Note
BlockNullReference is a valid return value.
When calling a sofs20 function of any layer, the version with prefix so should be used, not the one with prefix grp or bin.

Free all file blocks from the given position on.

The file block number (fbn) is the number of a block from the file (inode) point of view, 0 representing the first block, 1 the second, and so forth.
The data block number (dbn) corresponding to fbn is given by d[fbn]. Note, however, that array d[.] is only partially stored in the inode.

Precondition
ih is a valid handler of an inode in-use
Parameters
[in]ihinode handler
[in]ffbnfirst file block number
Note
A file can have holes, meaning references equal to BlockNullReference can appear between others that aren't.
Data blocks used to store references that become empty are also freed.
When calling a sofs20 function of any layer, the version with prefix so should be used, not the one with prefix grp or bin.

Read a file block.

Data is read from a specific file block of an in-use inode

If the referred file block has not been allocated yet, the returned data will consist of a byte stream filled with the null character (ascii code 0);

See also
soGetFileBlock
Precondition
ih is a valid handler of an inode in-use
buf is a valid pointer to a memory portion, at least BlockSize bytes long, and where the user has write permissions.
Parameters
[in]ihinode handler
[in]fbnfile block number
[in]bufpointer to the buffer where data must be read into
Note
When calling a sofs20 function of any layer, the version with prefix so should be used, not the one with prefix grp or bin.
